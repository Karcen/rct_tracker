# RCT 线下追踪调研管理系统

> 适配「广场随机人群 RCT 线下追踪调研」的桌面小程序。
> 纸质问卷 + 条码 + 扫码枪全流程管理，轻量化、易上手、免复杂部署。

中文 | [English](README.md)

## 一、项目简介

针对线下广场随机招募受试者的随机对照试验（RCT），本系统实现：

**现场登记 → 自动分配唯一条码 → 打印带条码纸质问卷 → 多轮随访用扫码枪核对身份与标记状态 → 台账管理与导出统计。**

系统**不做**问卷选项 OCR 识别，仅做**信息台账与配对管理**。所有数据保存在本地
`data/` 目录（单一 SQLite 文件 + 设置），可整体拷贝备份，无需安装任何数据库服务。

## 二、核心功能

| 模块 | 功能 |
| --- | --- |
| 信息登记 | 录入姓名 / 11 位手机号 / 性别 / RCT 分组；手机号强制校验；重复手机号预警；自动生成全局受试者 ID 与条码；条码预览、单张/批量打印（Code 128） |
| 扫码配对 | USB 扫码枪读码后自动弹出受试者信息与历史随访状态；手动标记问卷状态（正常有效 / 空白作废 / 信息不符 / 条码损坏 / 失访）；条码损坏时手动输入条码/ID 检索 |
| 数据台账 | 全字段台账；按 ID / 姓名 / 手机号 / 分组 / 轮次 / 状态多条件检索；编辑、重打条码、删除；一键导出 Excel（兼容 SPSS） |
| 统计分析 | 总人数、各组人数、各轮次有效 / 作废 / 失访等样本计数 |
| 系统设置 | 中英文切换；条码规则配置；数据库备份 / 恢复；打开数据目录 |
| 操作日志 | 登记 / 扫码 / 状态修改 / 导出 / 备份等行为留痕，可导出 CSV |

## 三、条码编码规则

```
条码 = 分组前缀 + 补零序号
  A = 干预组    →  A00001、A00002 …
  B = 对照组    →  B00001、B00002 …
受试者 ID（全局唯一）= S + 全局补零序号  →  S000001、S000002 …
```

- **一人一码，全程复用**：同一受试者多轮随访始终使用同一条码。
- 前缀与位数可在「系统设置」中修改（仅影响**之后**新登记的记录）。

## 五、运行环境

- 操作系统：**Windows 7/10/11**（macOS / Linux 亦可用于开发调试）
- Python：**3.8 及以上**（源码运行时需要；打包后的 exe 无需安装 Python）
- 硬件：普通 **USB 有线扫码枪**（免驱，识别为键盘输入）、常规桌面打印机

## 六、快速开始（源码运行）

### 1. 安装 Python
到 <https://www.python.org/downloads/> 下载 Python 3.8+，安装时**务必勾选
“Add Python to PATH”**。

### 2. 一键运行（推荐）
双击 **`run.bat`**，脚本会自动安装依赖并启动程序。

### 3. 手动运行
```bat
cd rct_tracker
python -m pip install -r requirements.txt
python main.py
```

## 七、打包为可双击运行的 EXE

双击 **`build.bat`** 即可（内部使用 PyInstaller）。完成后：

```
dist\RCT_Tracker.exe      ← 双击即可运行，无需安装 Python
```

将 `RCT_Tracker.exe` 拷贝到任意目录运行，会在其**同级目录**自动生成 `data\`
文件夹存放全部数据，便于整盘拷贝备份。

> 手动打包命令：
> ```bat
> pip install pyinstaller
> pyinstaller --onefile --windowed --name RCT_Tracker --collect-all barcode main.py
> ```
> `--collect-all barcode` 确保条码库自带字体被一并打包，**不可省略**。

本程序为跨平台 Python 代码，macOS 同样可用（`.bat` 是 Windows 脚本，Mac 改用 `.command`）。

## 八、使用说明

### 1）信息登记
1. 填写姓名、手机号（**必须 11 位数字**，否则禁止提交）、性别、分组。
2. 点击「保存登记并生成条码」：系统自动分配受试者 ID 与条码，并在右侧预览条码。
   - 若手机号已存在，会弹出**重复预警**，可选择是否继续。
3. 弹窗询问是否立即打印；也可在「最近登记记录」中选中任意一行后点「打印条码」。

### 2）扫码配对（多轮随访）
1. 顶部先选择**当前随访轮次**（第 1/2/3 轮）。
2. 将光标停留在「扫码 / 输入条码」输入框，用扫码枪扫描问卷上的条码。
3. 系统自动弹出该受试者的 ID、姓名、手机号、性别、分组及历史各轮状态。
4. 点击状态按钮完成标记：**正常有效 / 空白作废 / 信息不符 / 条码损坏 / 失访**。
   - 标记应用于「当前随访轮次」。误操作可点「撤销上一步」。
5. **条码损坏兜底**：直接在输入框手动输入条码或受试者 ID，点「查询」。

> 扫码枪等同键盘：扫码后会自动“输入”条码并回车，因此保持输入框聚焦即可连续作业。

### 3）数据台账
- 多条件检索（关键字 + 字段 + 分组 + 轮次 + 状态），双击行可编辑。
- 「导出 Excel」生成含 4 个工作表的台账文件（见下）。
- 「重新打印条码」可对选中（支持多选）批量补打条码。
- 「删除选中」前请先备份（不可撤销）。

### 4）统计分析
显示总人数、干预/对照组人数，以及各轮次的有效、作废、信息不符、条码损坏、
失访、未随访样本数量。点「刷新」更新。

### 5）系统设置
- 界面语言中英文切换（即时生效）。
- 条码规则：前缀、位数（仅影响新登记）。
- 数据备份 / 恢复、打开数据目录。

### 6）操作日志
查看并导出全部登记、扫码、状态修改、导出、备份记录（CSV，Excel 可直接打开）。

## 九、实操注意事项

### A. 条码打印
- 程序默认生成 **Code 128** 一维码 PNG（300 DPI，清晰、易识别），并通过浏览器
  打开**打印页**（自动弹出打印对话框，按 `Ctrl+P` 即可）。批量打印时多枚标签自动排版。
- **打印建议**：
  - 使用激光打印机或热敏标签机；喷墨打印请等墨干，避免条码模糊。
  - 打印比例选「100% / 实际大小」，**不要勾选“缩放以适应”**，以免条码失真。
  - 条码两侧需留白（程序已预留 quiet zone），切勿裁切到贴近黑条。
  - 打印后建议用扫码枪**试扫一张**确认可识别，再批量发放。
- Windows 下如需把条码图片直接发送默认打印机，可调用 `barcode_util.print_image_windows()`。

### B. 扫码枪接入
- 选购**USB 有线、免驱、HID 键盘模式**的一维码扫码枪（绝大多数廉价扫码枪均满足）。
- 插上即用，无需安装驱动。建议将扫码枪设置为**扫码后自动添加回车（Enter 后缀）**——
  多数扫码枪出厂默认即为此设置，若不是，可扫描说明书中的“添加回车后缀”设置码。
- 使用前在记事本里扫一张条码，确认能正确输出字符且自动换行。
- 作业时务必让光标停留在扫码输入框（切到「扫码配对」选项卡会自动聚焦）。

### C. 数据备份与恢复
- **实时保存**：每次登记、扫码标记均即时写入本地数据库，无需手动保存。
- **自动备份**：程序每次启动自动生成一次数据库快照（位于 `data\backups\`）。
- **手动备份**：「系统设置 → 立即备份数据库」。**现场作业建议每隔一段时间手动备份一次**，
  并把整个 `data\` 文件夹拷贝到 U 盘/云盘异地留存。
- **恢复/回滚**：「系统设置 → 从备份恢复」，选择某个 `.db` 备份即可回滚（恢复前会自动
  再备份当前数据，双重保险）。
- 迁移到另一台电脑：把 `data\` 文件夹整体拷贝过去覆盖即可。

## 十一、常见问题（FAQ）

- **Q：提示缺少条码依赖？** A：执行 `pip install python-barcode pillow openpyxl`，或直接双击 `run.bat`。
- **Q：界面中文乱码/方框？** A：系统缺中文字体。Windows 一般自带「微软雅黑」；如确实缺失请安装中文字体。
- **Q：扫码后没反应？** A：① 光标是否在扫码输入框；② 扫码枪是否设了回车后缀；③ 先在记事本验证扫码枪。
- **Q：条码扫不出来？** A：检查打印清晰度与缩放比例；用台账「重新打印条码」补打。
- **Q：手机号能否非 1 开头/少于 11 位？** A：系统强制 11 位数字，否则禁止提交（可在 `register_frame.py` 调整规则）。

## 十二、数据存储与隐私

- 数据**全部本地存储**，不联网、不上传，符合现场临时场地的隐私要求。
- 数据库文件：`data\rct_data.db`；备份：`data\backups\`；条码图：`data\barcodes\`；
  文本日志：`data\logs\`；导出文件：`exports\`。
- 受试者信息属敏感数据，请妥善保管备份介质，项目结束后按伦理要求处理。
